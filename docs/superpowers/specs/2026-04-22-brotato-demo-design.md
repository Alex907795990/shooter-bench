# 土豆兄弟 Demo 首版设计文档

- 日期：2026-04-22
- 状态：已确认，待执行
- 范围：首版单局制垂直切片

## 1. 目标

首版目标是做一个 `Brotato` 风格的单局制 demo，优先复刻玩法结构，而不是完整复刻数值、美术和全部内容量。玩家应能在一局内完成以下闭环：

`角色选择 -> 5 波战斗 -> 波间商店 -> 最终结算`

该版本应满足：

- 玩法结构接近原作，包含移动生存、自动攻击、敌人掉落材料、波间商店、武器与道具 build
- 数值和 UI 可以近似，但决策节奏必须成立
- 使用占位资源，不依赖正式美术
- 完整单局时长控制在 10-15 分钟

## 2. 首版范围

### 2.1 包含内容

- 1 个可选角色
- 5 个战斗波次
- 6-8 把武器
- 20-30 个道具
- 3-5 种怪物
- 完整的单局流程与结算
- 角色属性体系
- 武器自动攻击与武器槽
- 武器三合一升阶
- 敌人生成、掉落、拾取
- 商店刷新、购买、锁定、卖出、跳过
- 基础 HUD、商店界面、伤害数字、受击反馈

### 2.2 不包含内容

- 解锁系统
- 元进度、天赋树、难度等级
- 多角色
- Boss、精英、宝箱分支
- 手柄、移动端、联机
- 正式美术和正式音频资产
- 工程、元素、召唤等会显著扩大战斗分支的体系

## 3. 玩法还原原则

- 结构 1:1：必须保留“移动生存 + 自动攻击 + 波间商店 + build 成长”的核心循环
- 规则近似：商店、武器合成、属性修正、刷怪节奏保留原作味道，但允许首版简化数值
- 展示优先：即使资源占位，也必须保证 HUD、掉落、商店、受击反馈清晰可读
- 单局优先：所有成长都在一局内完成，不引入跨局系统

## 4. 单局流程

### 4.1 基础流程

1. 玩家进入角色选择场景
2. 确认角色后创建 run
3. 依次进行 5 波战斗
4. 每波结束进入商店
5. 第 5 波结束后进入结算

### 4.2 结束条件

- 失败：角色生命归零
- 成功：存活完成第 5 波

## 5. 波次与战斗规则

### 5.1 波次时长

首版波次时长按原作前短后长的结构设计：

- 第 1 波：20 秒
- 第 2 波：30 秒
- 第 3 波：40 秒
- 第 4 波：50 秒
- 第 5 波：60 秒

### 5.2 过波规则

- 倒计时归零立即过波
- 不要求清场
- 商店前结算本波材料与统计信息

### 5.3 战斗内成长

首版不做战斗内升级，不做暂停三选一。

敌人只掉落材料货币。玩家成长完全来自：

- 初始角色属性
- 武器购买与升阶
- 道具购买
- 商店中的 build 选择

### 5.4 玩家战斗方式

- 输入：`WASD` 移动
- 攻击：全自动
- 不做鼠标瞄准

## 6. 武器系统

### 6.1 武器槽

- 最多装备 6 把武器
- 武器自动攻击
- 商店可购买新武器或补齐同名武器做升阶

### 6.2 武器内容覆盖

首版 6-8 把武器至少覆盖以下攻击型：

- 近战挥砍
- 近战突刺
- 单发远程
- 高攻速低伤远程
- 散射远程
- 慢速高伤远程

### 6.3 武器关键字段

每把武器至少具备以下数据：

- 攻击间隔
- 基础伤害系数
- 攻击范围或射程
- 命中形态
- 目标选择规则
- 暴击或击退倾向
- 稀有度与价格

### 6.4 升阶规则

- 保留“三合一升阶”
- 同名同阶武器凑满时合成更高阶武器
- 升阶后继承更强数值和更高商店权重价值

## 7. 道具与属性系统

### 7.1 首版属性范围

首版道具与角色修正围绕以下属性展开：

- 伤害
- 攻速
- 暴击
- 生命上限
- 生命回复或吸血
- 移速
- 闪避
- 幸运
- 收获

### 7.2 设计目标

- 道具以被动数值修正为主
- build 至少能形成输出、生存、经济 3 类倾向
- 商店中“补武器体系”和“堆属性道具”必须形成取舍

## 8. 怪物与生成系统

### 8.1 怪物原型

首版 3-5 种怪物原型建议覆盖：

- 追击近战
- 高血慢速坦克
- 远程投射
- 快速骚扰或冲刺型

### 8.2 生成规则

- 每波有独立敌人预算
- 生成基于“波次预算 + 敌种权重 + 时间段曲线”
- 不使用写死的时间点脚本表
- 第 5 波通过混编密度形成高潮，不做 Boss

## 9. 商店系统

### 9.1 商店能力

首版商店支持：

- 购买
- 刷新
- 锁定
- 卖出武器
- 离开商店

### 9.2 商品结构

- 商店默认展示 4 个商品槽
- 商品来自武器池和道具池的加权随机
- 受稀有度和当前局内 build 轻度影响
- 锁定商品可保留到下一次商店

### 9.3 决策目标

商店必须让以下抉择成立：

- 现在补武器，还是补属性
- 现在刷新，还是保留材料
- 现在卖出旧武器，还是保留组合可能性

## 10. 场景与界面

### 10.1 场景拆分

- `Boot`：装配与启动
- `CharacterSelect`：角色确认
- `Battle`：战斗与 HUD
- `Shop`：波间商店
- `Result`：胜负结算

### 10.2 HUD 最低要求

HUD 必须持续展示：

- 当前生命
- 当前材料
- 当前波次
- 剩余时间
- 武器槽摘要

### 10.3 反馈最低要求

战斗与商店至少需要：

- 伤害数字
- 玩家受击反馈
- 材料掉落与拾取反馈
- 商品卡信息与购买结果反馈

## 11. 完成定义

首版满足以下条件才算完成：

- 能稳定跑完完整 5 波流程
- 每波结束都能稳定进入商店并返回下一波
- 敌人会生成、移动、攻击、掉落材料
- 玩家可拾取材料并在商店消费
- 武器会自动攻击，6 武器槽与三合一升阶可用
- 道具购买后能立刻影响战斗属性
- HUD 和商店信息对首次体验者可读
- 不存在阻断主流程的严重 bug
