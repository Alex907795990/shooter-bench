# Brotato 材料与波次小结设计文档

- 日期：2026-04-25
- 状态：已确认，待执行
- 范围：材料掉落、材料拾取、HUD 材料显示、战斗轮结束小结屏、商店占位入口

## 1. 目标

本次切片用于打通 `战斗 -> 材料收益 -> 波次小结 -> 商店占位` 的基础流程。

它不实现商店购买、商店刷新、下一波返回、最终整局胜负结算，也不引入道具或武器成长规则。它只负责让战斗收益能够被玩家看见、收集，并在波次结束后进入后续波间流程的入口。

## 2. 用户体验

玩家在战斗中击杀敌人后，敌人死亡位置会出现材料掉落。玩家靠近材料时自动拾取，HUD 中的材料数量实时增加。

战斗轮有倒计时。倒计时结束后，战斗停止并进入波次小结屏。小结屏展示本波获得材料、当前总材料、击杀数和剩余生命。

玩家点击小结屏的继续按钮后进入商店占位屏。商店占位屏只展示当前总材料和“商店后续实现”的提示，不提供购买、刷新或返回下一波功能。

## 3. 架构边界

所有会影响游戏结果的状态都属于跨帧逻辑状态，应放在 `InstanceContainer` 管理。表现层只读取这些状态并同步显示，不维护另一套会影响结算的材料、波次或击杀数据。

### 3.1 跨帧状态

需要新增或扩展以下逻辑状态：

- 当前波次状态：波次编号、剩余时间、是否已经结束
- 玩家经济状态：当前总材料、本波获得材料
- 本波统计状态：本波击杀数
- 材料掉落实例：位置、数量、拾取半径

这些状态通过对应的 `InstanceOps` 访问和修改，不在 `InstanceContainer` 上新增业务 CRUD 成员函数。

### 3.2 单帧事件

材料和波次流程需要以下事件语义：

- 敌人死亡事件触发材料掉落
- 材料被拾取事件用于表现层反馈
- 波次结束事件用于场景流转

事件只描述本帧发生的事实，不跨帧保存。需要长期存在的材料数量、波次状态和统计结果必须写入 `InstanceContainer`。

### 3.3 System 职责

新增或扩展的业务规则应放在 `System` 中：

- `WaveTimerSystem`：推进波次倒计时，倒计时归零后标记波次结束并产生波次结束事件
- `MaterialDropSystem`：读取敌人死亡事件，在死亡位置生成材料掉落
- `MaterialPickupSystem`：判断玩家与材料的距离，拾取后增加材料并移除掉落物
- 战斗 gating：波次结束后，敌人生成、武器发射、投射物结算和接触伤害不应继续推进新的战斗结果

`Resolver` 只负责导入输入、按固定顺序调度 `System`、输出并清理单帧数据，不直接实现波次或材料规则。

## 4. 表现层

### 4.1 Battle HUD

战斗 HUD 至少展示：

- 当前生命
- 当前总材料
- 当前波次
- 剩余时间

HUD 是逻辑状态的显示结果，不作为材料或波次数据源。

### 4.2 材料掉落表现

材料掉落使用简单占位图形即可。掉落实例存在时显示在战场中，被拾取或移除后从表现层同步删除。

### 4.3 波次小结屏

新增 `WaveSummary` 场景或等价界面，展示：

- 本波获得材料
- 当前总材料
- 本波击杀数
- 剩余生命
- 继续按钮

继续按钮进入商店占位屏。

### 4.4 商店占位屏

新增 `Shop` 场景或等价界面，展示：

- 当前总材料
- 商店后续实现提示

本次不实现购买、刷新、锁定、出售、离开商店或进入下一波。

## 5. 流程

基础流程如下：

1. 进入战斗场景
2. 波次倒计时开始
3. 敌人生成并参与现有战斗
4. 敌人死亡后生成材料掉落
5. 玩家靠近材料并自动拾取
6. HUD 实时显示材料变化
7. 倒计时归零
8. 进入波次小结屏
9. 点击继续
10. 进入商店占位屏

## 6. 非目标

本次不包含：

- 商店购买
- 商店刷新
- 商店锁定
- 武器或道具成长
- 多波循环
- 最终胜利或失败结果页
- 战斗内升级三选一
- 正式美术和音频
- 自动化测试

## 7. 完成定义

本次切片满足以下条件即视为完成：

- 敌人死亡会在死亡位置生成材料掉落
- 玩家靠近材料会自动拾取
- HUD 中材料数实时更新
- 波次倒计时归零后进入波次小结屏
- 小结屏显示本波材料、总材料、击杀数和剩余生命
- 点击继续进入商店占位屏
- 商店占位屏显示当前总材料和占位提示
- `npm run build` 通过
- 手动运行时主流程不被阻断
